# Changelog

## 1.2.0 - 2026-05-16

面向数据维护、注册稳定性和结果展示的小型版本更新。

### 更新亮点

- 新增隐藏结果通道，触发时使用专属结果卡展示，并从普通 Top 5 候选中隔离。
- 前端固定团体排序，数据集上传、识别范围和照片数量表的展示更稳定。
- 隐私提示和上传提示更明确，补充了遮挡、眼镜和清晰度对识别结果的影响说明。
- 特殊结果卡支持更醒目的超框展示效果。

### 修复与维护

- 修复特殊目录名在注册时被当作通配符展开的问题，按团注册会严格使用字面目录名。
- 注册脚本改为流式目录遍历，并在处理大图时自动缩放，降低内存占用。
- 注册时会跳过无脸、多脸或坏图样本，继续处理同一人剩余照片。
- `--band` 和 `--name` 的局部刷新流程保留已有特征，只更新目标条目。

### 运维提示

- 更新正式数据后继续使用 `python3 register.py --band <团名>` 刷新局部特征。
- 更新隐藏候选或普通候选后，都需要重启服务进程才能加载新的 `features.npz`。
- `exports/` 为本地分析导出目录，默认不进入版本控制。

## 1.1.0 - 2026-05-15

面向数据扩展和公开访问稳定性的更新。

### 更新亮点

- 新增 `millsage`、`dumbrock`、`mewtype` 三个团的入口，数据集上传和识别范围都能选择。
- 结果页从“相似度”改为更适合娱乐展示的“匹配度”，同时保留原始相似度用于排序。
- 注册工具支持 `python3 register.py --band 团名`，可以只刷新某个团的特征。
- 图片识别最大边恢复到 1280，默认 MTCNN 阈值调整为 `[0.6, 0.7, 0.7]`。
- 历史记录 JSON 损坏时会自动隔离坏文件，避免识别请求持续 500。

### 运维提示

- 新团要真正参与识别，需要先准备 `faces/<团>/<声优>/` 照片，再运行 `register.py --band <团>`。
- 更新 `features.npz` 后，重启 `seiyuumatch@3724` 和 `seiyuumatch@3725` 即可加载新特征。
- 公开部署模板已同步 `MAX_IMAGE_DIM=1280`。

## 1.0.1 - 2026-05-14

面向公开推广的体验优化版本。

### 更新亮点

- 识别请求支持排队，公开推广时更不容易因为同时访问而失败。
- 页面会显示排队、识别中和失败原因，用户不用只看到一句“识别失败”。
- 图片和图标加载更轻，首屏和结果页会更顺。
- 增加服务器部署模板和压测脚本，方便上线前检查承载情况。

### 部署建议

- 公开站点推荐使用两个服务进程加 nginx 分流。
- Cloudflare Tunnel 建议指向本机 nginx 入口。
- 推广前建议先跑 `load_test.py` 看延迟和失败率。

## 1.0.0 - 2026-05-14

SeiyuuMatch 第一个正式版本。

### Features

- 基于 MTCNN + AdaFace 的声优相似度识别 Web 应用。
- 支持上传图片识别多张人脸，并展示匹配声优头像、相似度和 Top 5 候选。
- 支持按团限制识别范围，默认启用 `mygo`、`avemujica`、`sumimi`。
- 支持默认阈值识别，以及“识别不到？降低阈值再试一次”的二挡检测。
- 支持独立头像目录 `avatar/<声优>/1.jpg` 和团标目录 `icon/<团>.png`。
- 支持数据集贡献上传，保存到 `faces_upload/` 等待人工审核。
- 支持反馈意见提交，保存到 `feedback/feedback.jsonl`。
- 支持隐私提示确认、左下角隐私提示入口、页面缩放控制和 GitHub 跳转。
- 支持 Cloudflare Tunnel 固定域名部署。

### Data

- 当前特征库包含 48 个声优条目。
- 修正 `志崎樺音` 名称。
- 支持 `佐々木李子` 同时归属 `avemujica` 和 `sumimi`。

### Operations

- 新增 `bench_env.py` 环境检测与性能测试脚本。
- README 增加服务器部署、systemd、Cloudflare Tunnel、性能测试说明。
- 上传识别图片限制为 6MB。
- 数据集上传单次限制为 80MB，`faces_upload/` 总量限制为 500MB。

### Notes

- `features.npz` 不是热更新，修改正式 `faces/` 数据后需要重新运行 `register.py` 并重启服务。
- 生产环境推荐让 Python 服务仅监听 `127.0.0.1:3724`，由 Cloudflare Tunnel 对外提供 HTTPS。
